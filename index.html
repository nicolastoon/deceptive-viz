Project 2 Persuasive or Deceptive Visualization?

Nicolas Toon — nitoon@ucsd.edu

Proposition: Which borough of New York City has the worst officers?

FOR the Proposition

Design Decisions and Rationale:

  • Choosing high-complaint officers per precinct as a metric (2): I chose to use the number of officers with 3+ substantiated complaints per precinct as a metric. While this measure may not be immediately intuitive to all audiences, it provides a strong representation of officer misconduct. I considered using the count of officers with 5+ complaints regardless of outcome, but ultimately decided that the current metric would be better. Since we are measuring substantiated complaints instead of just normal complaints, we are able to filter out officers who have received "petty" complaints.
  • Using a discretized sequential color scale (2): I used a sequential color scale because readers would intuitively interpret darker colors as worse outcomes. I also considered using a standard discretized color scale that increased the shade of the color linearly, but I decided to make the darkest color a bit darker than it normally would. This change will push readers to remember the boroughs with the highest values better.
  • Adding an annotation to further highlight message (2): The annotation is useful to put the colors into context. Since the color scale is discretized, it is hard to compare boroughs with each other. However, the text annotation draws attention and reinforces the message for the reader.
  • Purposefully creating large bins to mimic large variation (-1): The bins in the discretized colorscale are rather large, compared to the variance in the data points. For example, Staten Island, Manhattan, and Queens are all within the same bin with width 6, even though their numerical values are in very close proximity of each other. While the granularity of the visualization is reduced, I wanted to emphasize the gap between the Bronx and other boroughs.

AGAINST the Proposition

Design Decisions and Rationale:

  • Using diverging to differentiate between extremes (2): The cooler colors represented smaller values, and hotter numbers represented greater values. Since the human eye gravitates towards warmer colors, they are more likely to remember that Brooklyn has the greatest number of complaints, which is the message of this visualization. I decided that a diverging scale was more pleasant on the eye compared to a sequential scale, where all the colors are the very similar.
  • Using a continuous color scale (1): In this visualization, none of the boroughs have values that are particularly close, which makes the continuous color scale good because it helps the audience visualize how far apart each borough is from each other. You can see that Queens, Manhattan, and the Bronx are all close to each other on the color scale, while Staten Island and Brooklyn are on opposite sides. I was thinking about using a discretized scale, but decided that I wanted the audience to differentiate between each borough, not just the borough with the most complaints.
  • Choosing total number of complaints as the metric (-2): At first glance, this metric seems pretty appropriate for the question. More complaints generally equals worse police officers. However, the metric does fail to account for the number of police officers in each borough since the number of complaints will naturally be higher in areas where there are more police officers. There were other metrics that I tested out, like the proportion of complaints to precincts in each borough, but I decided to use the current metric because its results were more pronounced and is deceptive to the untrained eye.

Final Reflection

One thing that I found easy in this project was executing the design choice that I intuitively wanted to do and believed to the best. However, when it was time to create an opposing visualization, I found it difficult to go against my intuition and choose different design choice that communicated other ideas. For example, when manually making annotations or tweaking the colorscale so it was darker for certain regions, it went against my values to emphasize certain parts. If the data is actually significant, it should be easy to let the data speak for itself. Something I found surprisng was being able to differentiate persuasive and deceptive design decisions. The line between these is very blurry since one design choice might be persuasive in one visualization, but when applied to another visualization, loses it's value and instead hurts the message. For example, using a discretized vs continuous color scale was one design choice that I struggled to differentiate. In visualizations where the data isn't very closely packed, continuous color scales allow for the reader to grasp at the variance between the data. Whereas in visualizations where the data is closely packed, continuous color scales fall into the hole of that "just noticeable difference" and the pattern in the data is uninterpretable.

How do you now define “ethical analysis and visualization”? What bounds (if any) can you draw to distinguish “acceptable” persuasive choices vs. “misleading” ones (and if none, why not)? Ethical analysis and visualization is very difficult to achieve in the real world. It is impossible to filter or aggregate data that captures the whole picture. I believe that ethical visualization can only exist when you have many different visualizations trying to answer the same question. That way, a reader can take in all of the differences and truly think about what is causing these differences. When given a single visualization, it should be easy to determine which design choices were good, which would be better, etc. However, when given many different visualizations, it is impossible to determine which design choices are better than others. Each design choice has its own purpose. It might not suit one visualization, but it might suit another visualization, depending on the message and structure of the visualization.
